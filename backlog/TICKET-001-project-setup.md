# TICKET-001 — Initialize Next.js app (TypeScript, App Router)

**Status**: Open
**Depends on**: —

## Goal

Bootstrap the Next.js project with TypeScript and the App Router, establishing the directory structure described in the spec so all subsequent tickets can build on a consistent foundation.

## Acceptance Criteria

- [ ] `npx create-next-app` run with TypeScript, App Router, and Tailwind (optional) options
- [ ] Directory structure in place: `app/`, `components/`, `lib/`, `public/gtfs/`, `scripts/`
- [ ] `app/page.tsx` renders a placeholder (e.g. "Grenoble Tram Map")
- [ ] `app/layout.tsx` present with basic HTML shell
- [ ] `npm run dev` starts without errors
- [ ] `npm run build` completes without errors
- [ ] `.gitignore` excludes `node_modules/`, `.next/`, and `public/gtfs/*.json`

## Technical Notes

- Use `--app` flag (App Router) and `--typescript` when running `create-next-app`
- Node version: 18+ required for native fetch support used in Route Handlers
- `public/gtfs/` directory should exist but its JSON contents are generated by TICKET-002 and should not be committed
- No additional dependencies needed at this stage; Leaflet and react-leaflet are added in TICKET-006
